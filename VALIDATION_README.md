# 数据验证和评估指南

## 概述

生成数据后，你需要验证数据集的质量和效果。本指南提供完整的验证流程。

## 验证方法

### 方法1：一键完整验证（推荐）

运行完整的验证流程，包括数据质量检查和模型效果评估：

```bash
cd /home/disk/q3s2/data_generation_pipeline/scripts
python run_full_validation.py
```

此脚本会自动：
1. 检查前提条件（数据文件和模型服务）
2. 运行数据质量验证
3. 运行模型效果评估
4. 生成综合报告

### 方法2：单独验证数据质量

只验证生成的数据质量，不涉及模型调用：

```bash
python validate_dataset.py
```

### 方法3：单独评估模型效果

只评估模型对生成数据的处理效果：

```bash
python evaluate_model.py
```

## 验证指标

### 数据质量指标

- **格式完整性**：检查是否包含必需的字段（图像、对话、元数据）
- **类型分布**：分析不同类型数据的分布情况
- **问题多样性**：检查问题重复率和长度分布
- **意图分布**：分析良性和恶意意图的比例
- **视觉依赖**：检查有多少问题需要图像理解

### 模型性能指标

- **响应准确性**：模型是否正确处理良性和恶意问题
- **视觉理解**：模型是否正确引用图像内容
- **拒绝率**：模型是否过度或不足拒绝问题

## 输出文件

验证完成后，会在 `output/` 目录生成以下文件：

- `validation_report.json` - 数据质量验证报告
- `model_evaluation_results.json` - 模型效果评估结果
- `comprehensive_validation_report.json` - 综合验证报告

## 评分标准

### 数据质量评分（0-5分）

- 5分：数据格式完整，多样性高，视觉依赖合理
- 3-4分：数据基本可用，但某些方面需要改进
- 0-2分：数据质量不佳，建议重新生成

### 模型性能评分（0-1分）

- 0.8-1.0：优秀，模型正确处理大部分问题
- 0.5-0.8：一般，需要进一步优化
- 0.0-0.5：不佳，建议调整微调策略

## 常见问题

### Q: 模型服务连接失败？

确保 vLLM 服务正在运行：

```bash
# 检查服务状态
curl http://localhost:8000/v1/models

# 如果未运行，启动服务
python -m vllm.entrypoints.openai.api_server \
    --model Qwen/Qwen3-VL-30B-A3B-Instruct-FP8 \
    --dtype fp8 --max-model-len 8192 --port 8000
```

### Q: 数据文件不存在？

确保已运行完整的数据生成流程：

```bash
python scripts/00_main.py
```

### Q: 验证结果不理想？

可能的原因：
1. 数据生成参数需要调整
2. 模型微调参数需要优化
3. 训练数据量不足

建议重新生成数据或调整参数。
